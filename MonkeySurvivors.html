Op deze pagina ga ik vertellen over de game MonkeySurvivors
Die gemaakt is door Esat, Rinze, Alkan, Jorick en Simon

MonkeySurvivors is het product dat wij hebben gemaakt voor het project MicroGame.
De naam van het project zegt het al, je moet een simpel en klein spel maken voor op de telefoon.
Het doel van het project was om data van de speler te visualiseren en op te slaan en om goed te kunnen samenwerken in een groep.

Ons Idee

Ons idee was om een verslavende shooter te maken waarbij je wil blijven spelen om beter te worden.
Het spel van ons is gebaseerd op het spel Vampire Survivors. Dat betekent dat onze game een top-down 2D game is waarbij er steeds meer enemy's op je afkomen.
Het doel is om zo lang mogelijk te overleven. Hiervoor heb je natuurlijk items en wapens voor nodig.
Items kan je krijgen als je level up gaat door de paarse stenen op te pakken. Als je eenmaal level up bent kun je uit verschillende items kiezen:
Hier kan je items kiezen die je het makkelijker maken.
Er zijn meerdere items en elke keer als je level up gaat dan krijg je 3 random items waar je uit kan kiezen.
Voor de wapens hebben we ook een uniek systeem namelijk een crafting systeem. Dit is ook gelijk onze USP ( Unique Selling Point )
Deze moet je hebben want er moet een reden zijn waarom mensen eerder onze game zouden spelen dan Vampire Survivors.
De enemy's droppen materials zoals je links ziet. Deze kan je oppakken en hiermee kan je een wapen maken. Er zijn verschillende wapens die allemaal andere materials hebben:

Proces

Het proces ging heel goed. Aan het begin van het project werden de groepjes bekend gemaakt en mijn groepje was A3:
Esat, Rinze, Alkan, Jorick en Simon
Elk groepje had een SCRUM Master. Dit betekent dat diegene naar docenten kan voor vragen, alles moet inleveren en overzicht houden op de groep.
Esat was onze SCRUM Master en deed dit heel goed. Hij communiceert goed en was een goede Leider.
De eerste week was de concept fase. Hier mochten we nog niet werken in Unity. We hebben deze week goed geconcept met z'n allen waren op een leuk idee gekomen.
Omdat we al best snel alles voor mekaar hadden mochten we de eerste week al beginnen met het werken in Unity. Dit was erg fijn omdat we hierdoor voor het schema liepen.
De eisen waren om een verslavend telefoon spelletje te maken waarbij de progressie ( data ) wordt opgeslagen door middel van JSON. We moesten elke week User Stories, een Log en een Verbetervoorstel maken
Aan het einde van elke week was de voortgangspresentatie. We hebben afgesproken dat er elke week andere mensen gaan presenteren. Aan het eind van elke presentatie voelden wij ons wel goed omdat we voorliepen op de andere groepjes.
De fase van het werken aan de game was ook heel goed gegaan. We hadden de taken namelijk heel erg goed verdeeld:
-Esat ging aan de slag met de enemy's, XP, de wapens ( fixen ) en met JSON.
-Rinze maakte heel veel sprites voor de game en ging bezig met de geluiden en de promotievideo.
-Alkan heeft heel veel bugs opgelost maar ook heel veel gedaan voor de gameplay zoveel dat ik niet alles kan noem.
-Jorick heeft de map generation gemaakt en heeft gewerkt aan de crafting system.
-Ik (Simon) heb alle wapens / weapon system ( met hulp van Esat ) gemaakt en de items / item system werkend gemaakt
Dit groepje is het beste groepje dat ik ooit heb gehad bij een project omdat het heel goed ging met de samenwerking en de communicatie.
Dit kon je ook zien aan hoeveel we elke week af hadden gekregen en wat we hebben kunnen bereiken in dit project.
Uiteindelijk willen we proberen om de game na het project nog iets beter te maken zodat we heb op de Play Store ( Android ) kunnen zetten.
Dit willen we misschien doen met reclames en als het echt heel goed gaat dan willen we hem misschien ook nog uitbrengen op de App Store ( Apple )
